1. RESUM TEÒRIC
--------------------------------------------------------------------------------
- Lazy Evaluation (java.nio.file.Files.lines): Permet llegir arxius línia a línia 
  com un Stream, evitant carregar tot l'arxiu a la memòria RAM (OutOfMemoryError). 
  Ideal per a fitxers infinits o massius.
- Transformació de Dades (.map): Operador d'Stream que rep un element d'un tipus 
  (ex: String) i retorna un element d'un altre tipus (ex: Transaccio), creant 
  un pont entre el text brut i el Model de Domini.
- Gestió de Recursos (Try-with-resources): Ús de try(recurs) { ... } per assegurar 
  el tancament automàtic de connexions o arxius (I/O), independentment de si el 
  codi finalitza bé o llança una excepció. Protegeix contra "Memory Leaks" 
  (fuites de memòria).

2. CONTEXT D'APLICACIÓ I FUNCIONALITAT
--------------------------------------------------------------------------------
- Pipeline a ProcessadorBatch: S'ha dissenyat un mètode que llegeix dades, les 
  filtra, les instància i delega la validació a un motor extern, complint el 
  Principi de Responsabilitat Única (SRP).
- Programació Defensiva: Filtrar línies buides abans del .map() evita caigudes 
  fatals per NumberFormatException quan s'intenta parsejar text corrupte.
