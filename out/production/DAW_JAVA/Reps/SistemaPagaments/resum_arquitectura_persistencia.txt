1. CRONOLOGIA EVOLUTIVA DEL DESENVOLUPAMENT
--------------------------------------------------------------------------------
Durant aquesta sessió de disseny i refactorització, l'arquitectura de l'aplicació 
ha evolucionat des d'un sistema d'emmagatzematge local rígid fins a un motor de 
persistència empresarial multi-capa altament resilient:

• FASE 1 (Persistència en Fitxer): Es va implementar un 'TransaccioFitxerDAO' per 
  escriure cadenes de text en un fitxer .txt local. L'estratègia complia la funció, 
  però mancava d'escalabilitat, concurrència i capacitats de consulta indexada.
  
• FASE 2 (El Guardià Robust - Singleton): Es va dissenyar la classe 
  'ConnexioBDSingleton' amb constructor privat per centralitzar l'accés a MySQL. 
  La primera prova de foc es va fer amb el servidor apagat (XAMPP inactiu), on la 
  gestió defensiva de blocs 'try-catch' va capturar l'excepció 'Communications link 
  failure' correctament, evitant la caiguda catastròfica de l'aplicació.

• FASE 3 (Migració Relacional i Polimorfisme): Es va activar el servidor XAMPP i 
  es va crear la base de dades 'transaccions_db' juntament amb la taula. Es va 
  crear el 'TransaccioBDDAO' que aprofita el mètode 'obtenirDadesPersistencia()' 
  per injectar de forma transparent les dades de PayPal o Targeta a la BD.

• FASE 4 (Refactorització del Main i Neteja de Codi): Es va netejar el 'Main' 
  eliminant el codi de laboratori del Singleton, estructurant el flux de treball 
  en 4 fases clares (Inicialització, Recuperació, Execució Individual i Batch).

• FASE 5 (L'Auditoria del Try-With-Resources Niuat): Es va detectar que el 
  'try-with-resources' tancava automàticament la connexió global del Singleton. 
  Es va resoldre mitjançant un disseny de "Try Niuat" (Nested Try), on el bloc 
  extern captura les excepcions del Singleton i el bloc intern gestiona el 
  tancament automàtic exclusiu del PreparedStatement i el ResultSet.

2. ANÀLISI DETALLAT DE PATRONS D'ARQUITECTURA
--------------------------------------------------------------------------------

A) PATRÓ SINGLETON (Creacional)
Garanteix que una classe tingui una única instància a la memòria RAM del JVM i 
proporciona un punt d'accés global a ella.
• Implementació: Constructor 'private ConnexioBDSingleton()' que impedeix l'ús 
  de l'operador 'new' des de l'exterior. L'única via és el mètode estàtic 
  'getInstancia()', aplicant mandra en la inicialització (Lazy Initialization).
• Benefici: Estalvi dràstic de recursos de xarxa i processament al servidor MySQL, 
  evitant obrir un canal socket per cada transacció executada.

B) PATRÓ DAO - DATA ACCESS OBJECT (Estructural)
Separa completament la lògica de negoci de l'aplicació (el Processador de Pagaments) 
de la lògica d'accés a les dades (on i com es guarden).
• Implementació: Interfície abstracta 'TransaccioDAO' implementada per 
  'TransaccioFitxerDAO' i 'TransaccioBDDAO'.
• Benefici (Principi de Substitució de Liskov): Al 'Main' només cal modificar 
  l'instanciació del costat dret de la igualtat ('new TransaccioBDDAO()') i tot 
  el sistema canvia el seu comportament d'emmagatzematge instantàniament sense 
  alterar cap línia de codi de la lògica bancària.

C) PATRÓ STRATEGY & FACTORY METHOD (Comportament i Creacional)
• En el guardat: El DAO actua de forma polimòrfica. Crida a 't.getEstrategia()' 
  sense importar si és Targeta o PayPal, recollint una cadena estructurada 
  separada per punt i coma.
• En la lectura: S'ha implementat un mètode de factoria privat 'getTransaccio()' 
  que actua com un ORM (Object-Relational Mapper) artesanal. Llegeix el codi de 
  la columna 'tipus_pagament' ("PAYPAL" o "TARGETA"), trosseja les dades de 
  forma segura mitjançant '.split(";", 2)' i invoca el constructor de 
  l'estratègia concreta corresponent de forma encapsulada.

3. CONCEPTES DE JDBC, SEGURETAT I CODI SEGUR
--------------------------------------------------------------------------------

A) PREVENCIÓ D'ATACS D'INJECCIÓ SQL
Mai s'ha de fer servir la concatenació directa de text en cadenes SQL:
❌ String sql = "INSERT INTO tabla VALUES (" + variable + ")"; // COMPROMÈS!
L'ús de 'PreparedStatement' precompila la sentència SQL al motor de la base de 
dades utilitzant els marcadors d'interrogació '?'. Java envia les dades de manera 
aïllada al motor, de manera que qualsevol intent d'inserir codi SQL maliciós 
(per exemple, aprofitant el nom del titular de la targeta) és tractat estrictament 
com una cadena de text literal i neutralitzat automàticament.

B) EL RECEPTACLE DE RECURSOS (TRY-WITH-RESOURCES NIUAT)
Les connexions, sentències i conjunts de resultats de bases de dades mantenen 
canals oberts amb el sistema operatiu que provoquen fuites de memòria (Memory 
Leaks) si no es tanquen. La sintaxi 'try (Recurs r = ...) {}' assegura el seu 
tancament un cop finalitzat el bloc o si salta una excepció.

Com que tancar la 'Connection' destrueix el patró Singleton, l'arquitectura final 
correcta extreu la connexió a un 'try' pur sense parèntesis, i manté el control 
automatitzat només per a l'Statement i el ResultSet:

   try {
       Connection con = ConnexioBDSingleton.getInstancia().getConnection();
       try (PreparedStatement pstmt = con.prepareStatement(sql)) {
           // Execució segura de la query...
       } //pstmt es tanca automàticament aquí
   } catch (SQLException e) { ... } //con NO es tanca, queda lliure per al Singleton

4. ARQUITECTURA FINAL CONSOLIDADA (TransaccioBDDAO.java)
--------------------------------------------------------------------------------
A continuació queda registrat el codi de producció final que es va implementar 
i que va superar els tests d'integració de lectura i escriptura:

package Reps.SistemaPagaments.DAO;

import Reps.SistemaPagaments.Entitats.*;
import Reps.SistemaPagaments.Persistencia.ConnexioBDSingleton;
import java.sql.*;
import java.util.ArrayList;
import java.util.List;

public class TransaccioBDDAO implements TransaccioDAO {

    @Override
    public void guardarTransaccio(Transaccio t) throws PersistenciaException {
        String sql = "INSERT INTO transaccions (import, tipus_pagament, dades_pagament) VALUES (?, ?, ?)";
        try {
            Connection con = ConnexioBDSingleton.getInstancia().getConnection();
            try (PreparedStatement pstmt = con.prepareStatement(sql)) {

                String dadesCrues = t.getEstrategia().obtenirDadesPersistencia();
                String[] parts = dadesCrues.split(";", 2);
                String tipus = parts[0]; 
                String dades = parts[1]; 

                pstmt.setDouble(1, t.getImportTransaccio());
                pstmt.setString(2, tipus);
                pstmt.setString(3, dades);

                pstmt.executeUpdate();
                System.out.println(" >> OK: Transacció guardada a MySQL correctament.");
            }
        } catch (SQLException e) {
            throw new PersistenciaException(" >> ERROR: No s'ha pogut guardar a la BD: " + e.getMessage());
        }
    }

    @Override
    public List<Transaccio> obtenirTransaccions() throws PersistenciaException {
        String sql = "SELECT * FROM transaccions";
        List<Transaccio> historial = new ArrayList<>();
        try {
            Connection con = ConnexioBDSingleton.getInstancia().getConnection();
            try (PreparedStatement pstmt = con.prepareStatement(sql);
                 ResultSet rs = pstmt.executeQuery()) {

                while (rs.next()) {
                    double importTx = rs.getDouble("import");
                    String tipus = rs.getString("tipus_pagament");
                    String dades = rs.getString("dades_pagament");

                    Transaccio t = getTransaccio(tipus, dades, importTx);
                    historial.add(t);
                }
            }
        } catch (SQLException e) {
            throw new PersistenciaException(" >> ERROR: No s'ha pogut llegir l'historial de la BD: " + e.getMessage());
        }
        return historial;
    }

    private static Transaccio getTransaccio(String tipus, String dades, double importTx) {
        EstrategiaPagament estrategia = null;
        if (tipus.equals("PAYPAL")) {
            estrategia = new PagamentPaypal(dades);
        } else if (tipus.equals("TARGETA")) {
            String[] partsTargeta = dades.split(";");
            estrategia = new PagamentTargeta(partsTargeta[0], partsTargeta[1]);
        }
        return new Transaccio(importTx, estrategia);
    }
}
